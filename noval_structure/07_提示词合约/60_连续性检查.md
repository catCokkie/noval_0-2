# 60_连续性检查（协议增强版）

## 1. 任务目标
对章纲、场景、正文进行一致性与规则合规检查，并给出判级与回退建议。

## 2. 检查输入
```yaml
continuity_check_input:
  chapter_id: "v05-c173"
  chronology_system: "开拓历"
  time_anchor_ref: "K19-1849"
  host_faction: "西部董事会共和"
  event_package_tier: "B1-Heavy"
  is_key_chapter: "K3"
  cortex_decoy_action: ""
  deep_layer_action: ""
  system_blind_spot_signal: ""
  relation_changes: []
  resource_delta: {}
```

## 3. 必查清单
- 规模约束：
  - 卷规模是否在基线200章的±10%（180~220）。
  - 分段规模是否按剧情驱动并在基线20章的±10%（18~22）。
  - 字数是否匹配章节规格（small:2000~2200 / large:4000~4200）。
- 时间口径：是否使用开拓历主叙。
- 历史锚点：`1849/1850s` 是否仅存在于注释锚点而非章节主时间戳。
- R1/R2/R3：是否出现硬违规。
- F3：触发条件是否可解释。
- K3/K2：关键章密度是否异常。
- L2：关系变更是否登记。
- G1：资源变化是否登记。
- G4：临时组织是否有生命周期记录。
- W3：是否仅用于局势记录、未反推剧情。
- 认知对抗：是否登记 `cortex_decoy_action/deep_layer_action/system_blind_spot_signal`。

## 4. 输出模板
```yaml
continuity_check_output:
  chapter_id: "v05-c173"
  verdict: "pass|conditional_pass|fail"
  severity: "P0|P1|P2"
  findings:
    - id: "CHK-R1-01"
      level: "P0"
      issue: "无代价突破"
      fix: "补写代价并回填resource_delta"
  stats:
    f3_triggered: true
    key_chapter_density_window: "v05-c160~v05-c180"
    w3_record_only: true
    cognition_ops_registered: true
```

## 5. 回退建议
- `P0`：立即回退上游并阻断发布。
- `P1`：可条件通过，但需登记修复计划。
- `P2`：进入季度回归池。

## 6. 兼容性说明
上述新增字段仅为文档接口增强，不修改 `08_自动化/schemas/*.json`。
