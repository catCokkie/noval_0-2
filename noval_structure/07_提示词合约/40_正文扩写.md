# 40_正文扩写（协议增强版）

## 1. 任务目标
基于场景骨架写出章节正文，字数按章节规格执行，风格与设定与Canon一致。

## 2. 输入示例
```yaml
draft_expand_input:
  chapter_id: "v04-c089"
  chronology_system: "开拓历"
  time_anchor_ref: "K19-1849"
  host_faction: "南方庄园联邦"
  event_package_tier: "B1-Standard"
  is_key_chapter: "K2"
  chapter_size_mode: "small"
  word_count_min: 2000
  word_count_max: 2200
  cortex_decoy_action: ""
  deep_layer_action: ""
  system_blind_spot_signal: ""
  relation_changes:
    - "荣冠保守派 与 重建实用派：暂盟->敌对"
  resource_delta:
    sealed_threshold: "-1"
```

## 3. 输出要求
- 叙事必须遵守 `R1-R3`。
- 明确代价兑现，不得出现无代价突破。
- 若为 `K3/K2`，正文中必须出现对应关键章职责（转折/揭示/回收）。
- `W3` 相关内容仅作局势记录，不作为剧情强制因果。
- 章节主时间戳必须使用开拓历，`1849/1850s` 仅允许出现在注释锚点。

## 4. 推荐输出结构
```yaml
draft_expand_output:
  chapter_id: "v04-c089"
  actual_word_count: 2860
  is_key_chapter: "K2"
  text: "...正文..."
  chapter_end_hook: "单一钩子"
  time_anchor_ref: "K19-1849"
  cognition_ops:
    cortex_decoy_action: "..."
    deep_layer_action: "..."
    system_blind_spot_signal: "..."
  relation_changes:
    - "荣冠保守派 与 重建实用派：暂盟->敌对"
  resource_delta:
    sealed_threshold: "-1"
  compliance:
    r1: "pass"
    r2: "pass"
    r3: "pass"
```

## 5. 失败判定
- 字数不符合章节规格：`conditional_pass`。
  - `small`：<2000 或 >2200
  - `large`：<4000 或 >4200
- 关键章未体现关键职责：`fail`。
- 缺失关系/资源变更记录：`conditional_pass`。
