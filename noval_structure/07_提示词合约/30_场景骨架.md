# 30_场景骨架（协议增强版）

## 1. 任务目标
将章纲拆分为3~6个场景，保证每场景有明确功能，并保持可写成对应规格正文：
- 小章：2000~2200
- 大章：4000~4200

## 2. 输入字段
```yaml
scene_skeleton_input:
  chapter_id: "v03-c061"
  chronology_system: "开拓历"
  time_anchor_ref: "K19-1849"
  host_faction: "部落议会"
  event_package_tier: "B1-Standard"
  is_key_chapter: "none|K2|K3"
  cortex_decoy_action: ""
  deep_layer_action: ""
  system_blind_spot_signal: ""
  relation_changes: []
  resource_delta: {}
  chapter_size_mode: "small|large"
```

## 3. 输出模板
```yaml
scene_skeleton_output:
  chapter_id: "v03-c061"
  scene_count: 4
  scenes:
    - scene_id: "s1"
      function: "触发"
      objective: "抛出本章核心冲突"
      conflict: "谁阻止主角"
      cost_hint: "本场代价"
      transition_hook: "进入s2的钩子"

    - scene_id: "s2"
      function: "对抗"
      objective: "争夺信息/资源"
      conflict: "关系或势力反制"
      cost_hint: "关系折损"
      transition_hook: "进入s3"

  relation_changes:
    - "主体A 与 主体B：交易->敌对"

  cognition_ops:
    cortex_decoy_action: "表层顺从调度令"
    deep_layer_action: "深层执行反向接管"
    system_blind_spot_signal: "回执冲突与动作错位"

  resource_delta:
    steam_quota: "-5"
    trust_index: "-2"
```

## 4. 规则
- 至少1个场景体现 `R1` 代价。
- 至少1个场景承接 `R2` 信息路径（埋设或回收）。
- `event_package_tier=B1-Heavy` 时，冲突级别可升，但不可越权破坏硬规则。
- 若 `is_key_chapter=K3`，必须有“不可替代事件位”。
- 章节主时间戳必须使用开拓历，`1849/1850s` 仅允许出现在注释锚点。

## 5. 验收
- 场景功能不可重复堆叠。
- 结尾只保留单一钩子。
- 必须输出 `relation_changes` 与 `resource_delta`（可写“无变化”）。
- 可选输出 `time_anchor_ref` 与 `cognition_ops`，用于认知对抗回查。
