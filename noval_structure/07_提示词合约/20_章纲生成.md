# 20_章纲生成（协议增强版）

## 1. 任务目标
生成可执行章纲，满足：
- 对齐五卷长线节奏，但不强制固定每卷200章与每段20章。
- 先给出2~3个分段方案（多角度），再择优选1个执行。
- 选型后的卷/段规模需在基线10%浮动内。
- 为本章设定 `B1` 事件包、`FCT3` 主场势力、`K3/K2` 关键章标记。
- 输出可直接喂给 `30_场景骨架.md`。

## 2. 输入要求（新增可选字段）
```yaml
chapter_outline_input:
  chapter_id: "v02-c114"
  chronology_system: "开拓历"
  time_anchor_ref: "K19-1849"
  host_faction: "北方联邦政府"
  event_package_tier: "B1-Heavy"
  is_key_chapter: "K3"
  cortex_decoy_action: ""
  deep_layer_action: ""
  system_blind_spot_signal: ""
  relation_changes: []
  resource_delta: {}
  chapter_size_mode: "small|large"
  chapter_word_count_range: "small:2000~2200 | large:4000~4200"
```

## 3. 章纲输出模板
```yaml
chapter_outline_output:
  chapter_id: "v02-c114"
  objective: "本章唯一主目标"
  time_anchor_ref: "K19-1849"
  event_package_tier: "B1-Heavy"
  host_faction: "北方联邦政府"
  is_key_chapter: "K3"

  story_beats:
    - "起：局势触发"
    - "承：冲突升级"
    - "转：代价兑现"
    - "合：单一钩子"

  info_path_r2:
    setup: "本章埋设信息"
    reclaim_window: "v02-c130~v02-c168"

  cost_list_r1:
    - "能力副作用"
    - "关系/资源损失"

  foreshadow_ops:
    add: []
    advance: []
    reclaim: []

  relation_changes:
    - "主体A 与 主体B：暂盟->交易"

  cognition_ops:
    cortex_decoy_action: "林逸公开接受系统任务并提交顺从计划"
    deep_layer_action: "黑隼在关键节点执行反向拆解"
    system_blind_spot_signal: "动作回执延迟+目击证词冲突"

  resource_delta:
    steam_quota: "+8"
    intel_channel: "-1"

  hook:
    type: "悬念钩子"
    text: "仅一条"
```

## 4. 生成规则
- 单章仅保留一个主目标，避免并行主线。
- `W3` 评分只可写在“局势记录”，不得作为“必须发生”理由。
- 非关键章不得偷渡关键章信息密度。
- 每章新增术语<=1。
- 章节主时间戳必须使用开拓历，`1849/1850s` 仅允许出现在注释锚点。
- 字数规则必须绑定 `chapter_size_mode`：
  - `small` -> `2000~2200`
  - `large` -> `4000~4200`

## 5. 验收要点
- 字段齐全：`event_package_tier/host_faction/is_key_chapter/relation_changes/resource_delta`。
- 可选增强：`time_anchor_ref/cortex_decoy_action/deep_layer_action/system_blind_spot_signal`。
- 出现 `K3/K2` 时必须有“为何是关键章”的解释。
- 章纲必须声明 `chapter_size_mode`，并匹配对应字数区间。
