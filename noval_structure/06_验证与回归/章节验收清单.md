# 章节验收清单（增强版）

每章定稿前必须全部通过：

- [ ] 因果完整：事件由既有动机或前置线索触发。
- [ ] 规则合规：不违反 R1-R3 与观测规则。
- [ ] 角色一致：人物声线、动机、能力边界不漂移。
- [ ] 线索可追：关键线索可定位到具体段落或注释。
- [ ] 代价存在：本章至少一个可量化代价。
- [ ] 术语受控：新增术语不超预算且给出解释。
- [ ] 伏笔登记：埋设/回收已同步到账本。
- [ ] 钩子单一：结尾钩子类型唯一且明确。
- [ ] 成本完整：model_used/token_in/token_out/cost_usd 全记录。
- [ ] 关系日志登记：L2字段完整（from/to、状态、触发、代价、标签）。
- [ ] 资源变化登记：G1四资源变化已记录。
- [ ] 关键章标记合理：is_key_chapter 与实际内容一致。
- [ ] W3口径正确：局势记分仅作记录，不反推剧情。

## 判定规则

- 任一 `P0` 未修复 -> `fail`。
- 无 `P0` 且有 `P1` -> `conditional_pass`。
- 仅 `P2` 或无问题 -> `pass`。
