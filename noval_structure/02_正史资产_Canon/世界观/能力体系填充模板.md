# 能力体系填充模板（4维平衡版）

## 使用说明

1. 本模板只保留4个核心维度：`能力`、`代价`、`边界`、`信号`。
2. `触发`并入能力描述，`禁用`并入边界描述。
3. `反制`不在能力卡逐条填写，统一放在势力对抗文档或章节战术备注。
4. 非关键章可只填“能力+代价”；关键章（K2/K3）补齐4维。

---

## A. 全局参数（先填）

```yaml
ability_system_meta:
  chronology_system: "开拓历"
  core_mode: "4D(能力/代价/边界/信号)"

  gate_rules:
    - "R1-R3"
    - "B2+B1"
```

## C. 单序列卡模板（4维）

> 复制本段逐个填。每序列建议至少3个能力位。

```yaml
sequence_card:
  sequence_id: "SEQ_[势力]_[主/研]_[编号]"
  sequence_name: "[待填]"
  faction: "[北方/西部/部落/南方]"
  type: "[主序列/在研序列]"
  narrative_role: "[控场/侦查/爆发/封印/污染等]"

  ability:
    - name: "[能力位1]"
      desc: "[能力效果 + 触发条件]"
    - name: "[能力位2]"
      desc: "[能力效果 + 触发条件]"
    - name: "[能力位3]"
      desc: "[能力效果 + 触发条件]"

  cost:
    - metric: "[memory_noise/pollution_load/control_conflict/steam_quota/自定义]"
      delta: "[+1/-10 等]"
      note: "[代价表现]"

  boundary:
    hard_limit: "[不可越线规则]"
    chapter_limit: "[单章调用上限或频次]"
    gate_binding: "[与R1/R2/R3或B2/B1的绑定约束]"

  signal:
    visible: "[外显信号]"
    log: "[系统/设备日志信号]"
    audit_tag: "[用于章节验收的标签]"

  exposure:
    first_hint: "[卷-章范围]"
    first_use: "[卷-章范围]"
    reveal: "[卷-章范围]"
```
